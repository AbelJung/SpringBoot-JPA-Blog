-----------------------------------------------------------
	    
	    ** 스프링은 IOC (  =제어의 역전)인데
	    				사용자가 new하면  ( X )
	    				스프링이 자체적으로 new ( 0) 
	    				이유? 싱글톤 패던 + 레퍼런스 변수를 스프링이 관리 하려고
	    예)  Test라는 클래스가 있다고 가정하고
	    class A , class B가 있다고 가정했을 때
	    
	    class test {
	    		void hello(){
	    				A a = new A();
	    				//이렇게 선언하면 a는 hell 메소드 안에 있는 지역 변수고
	    				메소드를 호출시에 메모리에서 a가 떴다가 호출이 종료되면 메모리에서 사라짐
	    				a를 사용하려고 선언하면 계속 heap이 생김
	    				그래서 그냥 스프링이 알아서 관리해줄테니깐 필요할 때 갖다 쓰라=> 제어의 역전
	    				
	    		}
	    }
	    
	** 제어의 역전시 가장 먼저 componant scan을해서
	특정 패키지 이하를 전부다 스캔해서 필요한 것들을 메모리에 로드함(=IOC), 싱글콘 패턴으로 관리
	그래서 com.cos.blog로 처음에 패키지명을 만들면 이 패키지 이하의 것을 스캔하지
	다른 패키지명, 예를 들어 com.cos.abel이런식으로 변경해버리면 이 패키지는 사용 못함 ** 
	
	
	Run as => Spring Boot App클릭후 웹 기본 화면확인하자!	
	**여기서 포트를 8080으로 했을 경우 뭔가 겹쳐서 충돌이 되는지 정상적으로 실행이 안됨
